# trk

Generic work tracker - adapt to any domain via skills.

Short, fast, domain-agnostic.

## Install

```bash
# Install CLI
uv tool install ~/Code/trk

# Install trk-create skill
mkdir -p ~/.pi/agent/skills/trk-create
cp .agents/skills/trk-create/SKILL.md ~/.pi/agent/skills/trk-create/
```

## Usage

Use the `trk-create` skill to generate a domain-specific skill for your project:

```bash
/skill:trk-create
```

The skill asks domain questions and generates `.agents/skills/<domain>/SKILL.md` with domain-specific vocabulary.

## Generic Commands

```bash
trk init <target>                # Create new tracking target
trk show                         # View current state
trk new --desc "..."             # Create work item
trk try <id> "attempt"           # Record attempt
trk close <id> --conclusion      # Close work item
trk confirm <id> --conclusion    # Confirm work item
trk qadd --desc "..."            # Add to work queue
trk qdone <QID>                  # Mark queue item done
trk note "observation"           # Record note
trk grep "pattern"               # Search everything
```

## How It Works

1. **Generic infrastructure** (`trk` CLI) - domain-agnostic state management
2. **Domain skills** (generated by `trk-create`) - teach agent domain vocabulary
3. **Project-local skills** (`.agents/skills/<domain>/SKILL.md`) - loaded automatically by pi

Based on the [pi-autoresearch](https://github.com/davebcn87/pi-autoresearch) pattern:
- Generic extension/CLI provides tools
- Domain skills provide vocabulary and workflows
- Separation enables one tool, unlimited domains

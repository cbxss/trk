# track

Generic work tracker - adapt to any domain via project-local skills.

## Install

```bash
uv tool install ~/Code/track
```

## Usage

Use the `track-create` skill to generate a domain-specific skill for your project:

```bash
/skill:track-create
```

The skill asks domain questions and generates `.agents/skills/<domain>/SKILL.md` with domain-specific vocabulary and instructions.

## Generic Commands

```bash
track init <target>               # Create new tracking target
track show                        # View current state
track new --desc "..."            # Create work item
track try <id> "attempt"          # Record attempt
track close <id> --conclusion     # Close work item
track confirm <id> --conclusion   # Confirm work item (create confirmation)
track qadd --desc "..."           # Add to work queue
track qdone <QID>                 # Mark queue item done
track note "observation"          # Record note
track grep "pattern"              # Search everything
```

## How It Works

1. **Generic infrastructure** (`track` CLI) - domain-agnostic state management
2. **Domain skills** (generated by `track-create`) - teach agent domain vocabulary
3. **Project-local skills** (`.agents/skills/<domain>/SKILL.md`) - customize per project

Based on the [pi-autoresearch](https://github.com/davebcn87/pi-autoresearch) pattern:
- Generic extension provides tools
- Domain skills provide vocabulary and workflows
- Separation enables one tool, unlimited domains
